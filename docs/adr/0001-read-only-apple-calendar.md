# Read-only Apple Calendar sync for Events

Family App treats the shared Apple Calendar as the source of truth for Household Events and syncs it read-only in v1. For launch, Family App will consume a publicly shared Apple Calendar `webcal`/ICS feed rather than storing Apple credentials or using CalDAV. The feed URL is Household Calendar Connection data stored server-side, not a deployment environment variable; Parents may replace it from the Calendar workflow, but the client should not echo the full URL back after save. Calendar sync does not need to block the first production use of Chores, Goals, Points, and Rewards, but it remains part of the V1 production scope. This keeps calendar creation, edits, recurrence handling, and conflict resolution in Apple Calendar while letting Family App focus on household awareness, Participants, chores, goals, and rewards; the public feed URL should be treated as a server-side secret because anyone with the link can read the calendar feed.
