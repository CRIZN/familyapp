# Automatic Calendar Sync freshness

Family App saves a Parent-provided Family Calendar feed URL as server-side Calendar Connection data even when an immediate feed fetch fails, then attempts Calendar Sync on save, on Calendar page loads when the last sync attempt is more than 15 minutes old, and from scheduled Vercel cron. The scheduled route should rely on Vercel's cron protections first; add a custom server-only cron secret only if the deployment route would otherwise be reachable by ordinary web traffic. Calendar Sync keeps a rolling Event window from 30 days in the past through 180 days in the future; synced Events outside that window, or retained-window Events that disappear from the Family Calendar feed, are removed from Family App with their Event Enrichment because Apple Calendar remains the source of truth. Synced Events are matched by External Event Identity: Apple/iCalendar UID for single Events and UID plus occurrence start time for recurring Event occurrences. Calendar Sync preserves Event Enrichment when the External Event Identity remains the same, even if Apple changes the Event title, time, or location. Replacing the feed URL treats the new feed as a new Calendar Connection source: existing synced Events and their Event Enrichment are cleared before syncing the new feed. New synced Events default to all-Household and Calendar Sync does not infer Participants from Event text in v1; Parents manually tag Participants through Event Enrichment. Parents may trigger a safe manual Sync Now action from the Calendar workflow for troubleshooting; Children only see the resulting Agenda. Failed sync attempts keep the last successfully synced Events visible and surface a safe warning with the last attempted sync time and Sync Now action rather than clearing the Agenda. All-Day Events remain distinct from timed Events and appear above timed Events for their date in the Agenda. The client must not echo the full feed URL after save; Parents see only safe metadata such as connection state, last sync attempt, last successful sync, event count, and non-secret failure status. This makes Calendar Sync feel automatic without introducing realtime infrastructure or unbounded calendar history storage, while keeping the public feed URL treated as a secret.
