# Parent accounts and Child PINs

Family App will use real Parent accounts and Parent-managed Child PINs for v1 access. Parent accounts will launch with Supabase email magic links rather than passwords, and access will be allowlisted by exact Parent email stored on the Household before or during login. Every Parent request must verify that the authenticated Supabase user's email maps to an allowed Parent row before returning or mutating Household data. Children will use 30-day signed, httpOnly, same-site cookies created from Parent-managed PINs; each cookie should contain only the Household and Child identity needed to validate the session server-side. Child PIN changes should invalidate existing Child sessions by comparing a session version or PIN update timestamp in the cookie to the current Child row. A PIN authenticates a Child only inside an already established Household context; a bare PIN must never discover Household data or Child profiles. This avoids email/password accounts for Children while still keeping each Child's Chores, Goals, Point Balance, and Reward Requests separate inside the Household.
